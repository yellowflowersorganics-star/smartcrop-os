## ğŸ’» Development Setup

### Prerequisites

Install these on your machine:

```bash
# Node.js 18+
# Download from: https://nodejs.org

# Git
# Download from: https://git-scm.com

# Docker & Docker Compose
# Download from: https://www.docker.com

# VS Code (recommended)
# Download from: https://code.visualstudio.com
```

### Step-by-Step Setup

**1. Clone Repository**

```bash
git clone https://github.com/your-org/cropwise.git
cd cropwise
```

**2. Backend Setup**

```bash
cd backend
npm install
cp .env.example .env
# Edit .env with your settings
npm run migrate
npm run dev
```

**3. Frontend Setup**

```bash
cd frontend
npm install
cp .env.example .env
# Edit .env with your settings
npm run dev
```

**4. Access Application**

- Frontend: http://localhost:5173
- Backend API: http://localhost:8080
- API Docs: http://localhost:8080/api-docs

**5. Run Tests**

```bash
# Backend tests
cd backend
npm test

# Frontend tests
cd frontend
npm test
```

### VS Code Extensions

Install these recommended extensions:

```
- ESLint
- Prettier
- GitLens
- Docker
- Thunder Client (REST client)
- Error Lens
```

### Troubleshooting Setup

**Issue: Port already in use**

```bash
# Find process using port 8080
lsof -i :8080
# Kill process
kill -9 <PID>
```

**Issue: Database connection failed**

```bash
# Make sure PostgreSQL is running
pg_isready

# Or use SQLite for development (default)
# Check DB_DIALECT in .env
```

**Issue: Module not found**

```bash
# Clear and reinstall
rm -rf node_modules package-lock.json
npm install
```

If you're still stuck, ask in #dev-help channel!

## ğŸ†˜ Getting Help

### When You're Stuck

**1. Try to solve it (15-30 min)**
- Search documentation
- Google the error
- Check Stack Overflow

**2. Ask your onboarding buddy**
- Quick questions
- Clarifications
- General guidance

**3. Ask in #dev-help**
- Technical issues
- Code questions
- Tool problems

**4. Schedule 1:1 with senior dev**
- Complex problems
- Architecture questions
- Design discussions

### How to Ask Good Questions

**Bad:**
> "It doesn't work. Help!"

**Good:**
> "I'm trying to create a new batch via the API, but I'm getting a 500 error. Here's my request:
> 
> ```json
> POST /api/batches
> {
>   "batchCode": "BATCH-001",
>   "farmId": 1
> }
> ```
> 
> Error: `Internal Server Error`
> 
> I checked the logs and saw: `TypeError: Cannot read property 'id' of undefined`
> 
> I've tried:
> - Verified farmId exists
> - Checked API documentation
> - Tested with Postman
> 
> Any ideas what might be wrong?"

**Include:**
- What you're trying to do
- What you've tried
- Error messages
- Code snippets
- Screenshots (if helpful)
